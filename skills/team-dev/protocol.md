# 协作协议 (Collaboration Protocol)

## 消息传递规则

### 1. 广播消息 (Broadcast)

**使用场景：**
| 场景 | 触发条件 | 消息类型 | 内容要求 |
|-----|---------|---------|---------|
| 项目启动 | 团队创建完成 | INFO | 项目目标、成员名单 |
| 里程碑达成 | 阶段性任务完成 | SUCCESS | 成就描述、感谢 |
| 紧急通知 | 重大问题发现 | URGENT | 问题描述、影响范围 |
| 代码冻结 | 进入测试阶段 | WARNING | 冻结时间、禁止操作 |
| 项目完成 | 所有任务验收 | COMPLETE | 成果总结、经验分享 |

**广播模板：**
```json
{
  "type": "broadcast",
  "category": "milestone|urgent|warning|info",
  "content": "消息内容",
  "summary": "一句话摘要"
}
```

**示例：**
```
【里程碑】围棋游戏项目教程系统已完成！
- 完成内容：教程列表、详情页、章节导航
- 贡献者：Coder
- 下一步：复盘分析功能
```

---

### 2. 私聊消息 (Private Message)

**使用场景：**

| 场景 | 发送方 | 接收方 | 内容 |
|-----|-------|-------|-----|
| 任务分配 | PMO | Coder | 任务详情、截止时间 |
| 进度汇报 | Coder | PMO | 完成度、阻塞点 |
| 技术咨询 | Coder | Architect | 问题描述、上下文 |
| 代码审查 | Architect | Coder | 审查意见、改进建议 |
| Bug报告 | Tester | Coder | 重现步骤、预期vs实际 |
| 测试确认 | Coder | Tester | 测试用例、通过情况 |

**私聊模板：**
```json
{
  "type": "message",
  "recipient": "role-name",
  "content": "详细消息",
  "summary": "简短摘要",
  "taskId": "任务ID (可选)"
}
```

## 任务流转协议

### 任务生命周期

```
[待办] → [进行中] → [待验收] → [已完成]
   ↑                                    |
   |_____________________________________|
              (打回重做)
```

### 状态变更规则

| 当前状态 | 触发操作 | 目标状态 | 触发者 |
|---------|---------|---------|-------|
| 待办 | 开始工作 | 进行中 | 任务负责人 |
| 进行中 | 遇到阻塞 | 进行中(阻塞) | 任务负责人 |
| 进行中 | 完成开发 | 待验收 | 任务负责人 |
| 待验收 | 验收通过 | 已完成 | PMO |
| 待验收 | 打回修改 | 进行中 | PMO/Architect |

## Mailbox 同步规则

### 同步时机

**必须同步：**
1. 任务状态变更时
2. 发现阻塞或风险时
3. 需要其他角色协助时
4. 交付物完成待验收时

**可选同步：**
1. 阶段性进展汇报
2. 技术方案讨论
3. 临时问题咨询

### 同步内容模板

```markdown
## 同步消息

**类型:** 进度汇报 / 阻塞报告 / 请求协助 / 交付验收

**任务ID:** T-XXX

**当前状态:** [状态]

**内容:**
- 完成情况: xxx
- 阻塞因素: xxx (如有)
- 需要协助: xxx (如有)
- 下一步计划: xxx

**优先级:** P0/P1/P2/P3
```

## 冲突解决协议

### 1. 技术争议
```
1. 相关方私聊讨论
2. 无法达成一致 → 升级至 Architect
3. Architect 裁决 → 执行裁决结果
```

### 2. 任务冲突
```
1. PMO 评估优先级
2. 高优先级任务优先
3. 资源冲突 → PMO 协调
```

### 3. 质量争议
```
1. Tester 提供证据
2. Coder 响应解释
3. PMO 仲裁决定
```

---

## 响应时间协议

| 消息类型 | 响应时间 | 超时处理 |
|---------|---------|---------|
| URGENT (紧急) | 5分钟内 | 自动升级至广播 |
| HIGH (高优先级) | 30分钟内 | 发送提醒 |
| NORMAL (普通) | 2小时内 | 下一轮同步处理 |
| LOW (低优先级) | 24小时内 | 汇总处理 |

## Checkpoint 检查点

### 每日站会 (Daily Standup)
```json
{
  "type": "broadcast",
  "time": "每日固定时间",
  "content": [
    "昨日完成",
    "今日计划",
    "阻塞问题"
  ]
}
```

### 周会 (Weekly Review)
```json
{
  "type": "broadcast",
  "time": "每周五",
  "content": [
    "本周进度总结",
    "下周计划",
    "风险评估",
    "改进建议"
  ]
}
```

### 里程碑评审 (Milestone Review)
```json
{
  "type": "broadcast",
  "trigger": "里程碑完成",
  "content": [
    "交付物清单",
    "验收标准",
    "遗留问题",
    "下一步计划"
  ]
}
```

## 消息格式规范

### 消息头 (Message Header)
```yaml
---
type: broadcast|message
priority: P0|P1|P2|P3
category: milestone|urgent|warning|info
timestamp: ISO8601
sender: role-name
---

### 消息体 (Message Body)
**主题:** xxx

**详细内容:**
1. xxx
2. xxx

**相关任务:** T-XXX (可选)
**附件:** xxx (可选)
```

### 消息尾 (Message Footer)
```
---
发送时间: 2024-02-12T10:00:00Z
期望响应: [时间]
联系渠道: Mailbox
```
